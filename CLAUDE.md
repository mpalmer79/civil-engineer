# CLAUDE.md

This file contains standing instructions for Claude Code when working in this repository.

## Project Identity

Product name: Civil Engineer AI
Repository name: civil-engineer
Demo project: Brookside Meadows

Do not rename the product, repository, or demo project.

Forbidden alternate names:

* CivilSite AI
* Cedar Ridge
* Any other project or demo subdivision name

## Professional Boundary

Civil Engineer AI is a review-support and evidence-organization system.

It does not:

* Approve plans
* Certify compliance
* Stamp drawings
* Replace a licensed Professional Engineer
* Verify CAD drawings
* Validate final design
* Make final engineering decisions
* Declare a project safe
* Declare a project fully compliant

Use careful review-support language.

Allowed wording examples:

* needs human review
* review-support finding
* potential issue
* evidence link
* reviewer action
* consistency concern
* needs follow-up
* needs more information
* reviewer checked

Forbidden status or action wording:

* approved
* certified
* verified
* passed
* failed
* compliant
* noncompliant
* safe
* unsafe
* design validated

Do not create an action named approve.

## Attribution and Footer Rules

Do not add tool attribution, generated-by text, session links, AI disclosures, or co-author trailers anywhere in this repository.

This applies to:

* PR summaries
* Commit messages
* Markdown files
* README files
* Documentation
* Code comments
* Test comments
* Final response summaries
* Changelog entries

Forbidden examples:

* Generated by Claude Code
* Generated with Claude Code
* Claude session links
* claude.ai/code/session URLs
* Co-Authored-By trailers
* AI-generated footers
* Any tool signature or attribution line

Before completing any task, search changed files, documentation, commit messages, PR text, code comments, and final summaries for these strings and remove them, except where CLAUDE.md intentionally lists them as forbidden examples (the occurrences in this file are the allowed exception):

* Generated by
* Claude Code
* claude.ai/code
* Co-Authored-By
* AI-generated

## Development Scope Rules

Keep live AI calls disabled by default.

Do not add any of the following unless explicitly requested in a future phase:

* Real PDF parsing
* DWG parsing
* DXF parsing
* Autodesk integration
* GIS integration
* OCR
* Computer vision
* Vector search
* Authentication
* Deployment setup
* External paid APIs

Seeded data is acceptable when clearly labeled as seeded review-support data.

Do not claim that seeded data was extracted from real CAD, PDF, GIS, or plan files.

## Writing Style Rules

Do not use em dashes in prose, comments, documentation, commit messages, PR descriptions, or final summaries.

Use direct, professional language.

Avoid marketing exaggeration.

Avoid claims that exceed the system's actual capabilities.

## Code Quality Rules

Follow the existing repository style.

Keep naming consistent with previous phases.

Avoid overengineering.

Do not remove existing working phase functionality unless explicitly instructed.

Add meaningful tests for new backend behavior.

Keep frontend types aligned with backend schemas.

Prefer backend-canonical data over browser-simulated data for implemented features.

## Completion Checklist

Before finishing any task:

1. Confirm no forbidden attribution text was added.
2. Confirm no Claude Code footer or session link appears anywhere.
3. Confirm no prohibited engineering decision language was introduced.
4. Confirm no em dashes were added.
5. Confirm new functionality is documented.
6. Confirm relevant tests were added or updated.
7. Confirm existing functionality was not removed.
