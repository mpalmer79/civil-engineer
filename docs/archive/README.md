# Archived Documentation

The documents in this directory are historical, non-authoritative snapshots.
They are kept for provenance: phase and sprint records, earlier roadmap
variants, recruiter-oriented walkthroughs, superseded specifications, and the
per-feature and per-API notes that were folded into the canonical documentation
set during the documentation consolidation.

These files are not maintained and may not reflect the current system. Durable
decisions from them were carried into the canonical documents before archiving.
For current information, use the canonical set:

- `docs/PRODUCT.md`: what the product does, who it is for, capability boundary.
- `docs/ARCHITECTURE.md`: system architecture and subsystem design.
- `docs/DOMAIN_MODEL.md`: the review-support domain model.
- `docs/SECURITY.md`: security posture and the professional boundary.
- `docs/OPERATIONS.md`: release readiness, pilot operations, billing, email.
- `docs/DEPLOYMENT.md`: Railway deployment, database, observability.
- `docs/TESTING.md`: test suites, coverage, and release gates.
- `docs/API.md`: API surfaces, deferring the contract to the OpenAPI schema.
- `docs/DXF_VALIDATION.md`: the DXF proof methodology and artifacts.
- `docs/REFERENCE_PROJECT.md`: the synthetic Brookside Meadows reference project.
- `docs/ROADMAP.md`: committed, planned, research, deferred, and out-of-scope work.

Decision records live in `docs/adr/`. The documentation map is `docs/README.md`.

Nothing here should be cited as the current state of the product.
