# DXF Proof of Concept

Public page: `/proof-of-concept` (redirects from `/proofofconcept` and `/poc`)

## Purpose

Prove, through reproducible evidence, that Civil Engineer AI can ingest a
structurally valid civil-site DXF, extract useful metadata, identify
references and review-support observations, and organize results for human
review.

The proof demonstrates deterministic DXF metadata extraction, consistency
checking, reference identification, and reviewer-support analysis. It does
not demonstrate engineering correctness, design validation, compliance, or
approval of any kind.

## The synthetic drawing

`brookside_meadows_realistic_submission.dxf` is generated by
`scripts/generate_brookside_proof_dxf.py`. It is a conceptual subdivision
plan for the synthetic Brookside Meadows case study in the Town of Hartwell:

* 47 conceptual lots with lot number labels
* road alignment with a cul-de-sac (polyline, arc, circle)
* grading contours (polylines and splines) with elevation labels
* wetland boundary and 50 ft buffer
* detention basin (outline plus hatch) and infiltration basin
* storm pipes, an outfall point, erosion controls, utility corridors
* facility labels including DETENTION BASIN 1 and INFILTRATION BASIN 1,
  which intentionally share the number 1 without being the same facility
* three intentional naming inconsistencies that exercise the three
  facility-conflict rules
* sheet references C-3.1 and C-5.0, detail reference DETAIL 4/C-5.0, an
  intentionally missing sheet C-9.9, and an intentionally ambiguous
  DETAIL ?/C-4.X
* five reusable blocks and a paper-space title layout
* three intentionally unclassifiable layers
* declared units (feet) in the DXF header

The geometry is conceptual. It is not surveyed, not georeferenced, and not
suitable for construction.

## Baseline results

The verified run produces exactly:

| Metric | Value |
| --- | --- |
| Upload HTTP result | 200 |
| Intake validation | accepted |
| Parse status | completed |
| Entities | 168 |
| Layers | 15 |
| Blocks | 5 |
| Text records | 80 |
| Reference candidates | 16 |
| Review-support findings | 8 |
| Matched references | 3 |
| Missing or ambiguous references | 2 |

These values live in `scripts/dxf_proof_expected.json` (the versioned ground
truth) and in the generated
`public/proof-of-concept/dxf/brookside_meadows_dxf_test_result.json`. The
page renders from the generated artifact, never from copies of the numbers,
and fails visibly if the artifact and displayed metrics disagree.

## Supplied artifacts note

The originally supplied proof artifacts (from `/mnt/data/civil_dxf_test/`)
were not available in the build environment. The committed artifacts were
regenerated from the known synthetic Brookside scenario by the deterministic
harness described in `DXF_TEST_METHODOLOGY.md`. No claim is made that the
original binary files were used.
